记忆跃迁与增益

M0
空白状态
触发语：帮我女朋友选个礼物
增益：无。系统只能追问预算、喜好、禁忌，用户费力度高。
M0 → M1
新增：给女朋友选礼物预算在 1000 元以内
触发语：我的预算在 1000 块以内，喜欢紫色
增益：后续推荐不再追问预算，自动过滤超预算选项。
M1 → M2
新增：女朋友喜欢紫色
同一句触发，但主体归属不同：预算属于用户，紫色偏好属于女朋友。
增益：推荐能主动命中颜色偏好，减少用户纠错。
M2 → M3
新增：用户希望只保留一个最合适推荐
触发语：你推荐的有点多，帮我就保留一个吧
增益：后续推荐直接收敛成一个方案，不再输出长列表。

展开费力度评分规则
重复说明成本
是否反复说明预算、喜好、格式。
澄清成本
Agent 还需要追问多少关键缺口。
修正成本
用户是否需要纠正主体归属或输出过多。
控制成本
用户能否查看、删除、更新记忆。
