# 参赛验证测试样例

本文档整理当前分支已经确认可跑通的验证样例，覆盖 Workbench 和 Agent Skill 两条路径。

验证目标：

- 基础场景：复用 Workbench 当前预置的 4 个测试场景，顺序与前端一致。
- 复杂礼物场景：验证记忆加入、更新、删除、管理，以及跨 Session 召回已选礼物。
- Workflow 自学习：验证用户教给 Skill 的交互规则会沉淀为 workflow 记忆，并在后续短 query 中生效。

最近一次本地验证：

- Workbench：使用 MiniMax 真实 LLM，在 `http://127.0.0.1:8787` 通过礼物选择、跨 Session 召回、销售渠道 workflow 验证。
- Agent Skill：使用 Direct Skill runtime + local JSON 临时目录，通过 `memory-write` 和 `memory-pack` 验证同一套记忆写入与召回逻辑。

## 使用方式

### Workbench 路径

1. 启动 Workbench。
2. 进入 `Agent Chat`。
3. 按右侧“比赛测试案例”中的步骤逐条点击或复制输入。
4. 遇到 `Eval Round` 步骤时运行真实 LLM Eval。
5. 遇到 `Round 2 / Round 3` 的第一句话时，新开 Session 后再输入该句，Session 是 Eval 的最小单元。
6. 在 `History Evals` 查看多轮评分、费力度、记忆节省信息点和详情。

### Agent Skill 路径

1. 在支持 Skill 的 Agent 中安装并触发 `$assist-everything-betterandbetter-skill`。
2. 首次触发时确认记忆授权和当前记忆后端。
3. 后续直接自然对话，不需要每轮都带 `$skill` 前缀。
4. 记忆管理可用自然语言，例如：
   - `展示当前记忆`
   - `删除 她喜欢紫色`
   - `清除当前记忆`
   - `退出 skill`

## 一、基础场景：Workbench 当前预置 4 个场景

以下 4 个场景按当前 Workbench 前端展示顺序整理，步骤文本保持一致。

### 1. 女朋友生日礼物

模块：复杂送礼协作记忆

| 步骤 | 标签 | 输入 / 动作 |
| --- | --- | --- |
| 1 | Round 1 任务 | 帮我给女朋友选个生日礼物。 |
| 2 | 形成记忆 | 预算1000元左右；她喜欢紫色；如果是首饰，她喜欢玫瑰金；以前送过玫瑰金项链，送过的不要再送。 |
| 3 | 展示记忆 | 展示当前记忆。 |
| 4 | Eval Round 1 | 先评估当前 session 是否正确形成偏好、历史和决策。 |
| 5 | Round 2 推荐 | 那再帮我给女朋友选生日礼物做一个推荐。 |
| 6 | 更新约束 | 不是，我想换个非首饰品类。 |
| 7 | Eval Round 2 | 评估第二个 session 是否减少重复说明。 |
| 8 | Round 3 推荐 | 那再帮我给女朋友选生日礼物做一个新的推荐。 |
| 9 | 删除后复测 | 删除 她喜欢紫色。然后：再给一个不重复的礼物方向。 |
| 10 | Eval Round 3 | 评估删除后是否不再使用紫色。 |

预期观察：

- Round 1 形成预算、颜色、首饰材质偏好、已送历史。
- Round 2 不要求用户重复预算和已送历史，并能接收“非首饰”更新约束。
- Round 3 删除紫色后，不再把紫色作为推荐条件。

### 2. 家庭旅行规划

模块：场景规则与条件化更新模块

| 步骤 | 标签 | 输入 / 动作 |
| --- | --- | --- |
| 1 | Round 1 任务 | 帮我安排北京周末 2 天亲子旅行。 |
| 2 | 形成记忆 | 以后家庭出行请记住：父亲膝盖不好，步行要少；孩子喜欢自然和动物；我不喜欢人挤人的网红点。 |
| 3 | 展示记忆 | 展示当前记忆。 |
| 4 | Eval Round 1 | 先评估当前 session，再进入下一轮。 |
| 5 | Round 2 应用 | 帮我安排杭州 3 天家庭行程。 |
| 6 | 更新/条件化 | 这次父亲不去，只有我和孩子；避开网红点这条保留，但少步行不适用。 |
| 7 | Eval Round 2 | 评估第二个 session 的费力度和记忆复用。 |
| 8 | Round 3 复用 | 帮我安排上海 1 天亲子自然路线。 |
| 9 | 删除后复测 | 删除孩子喜欢动物这条记忆。。然后：安排南京半日游。 |
| 10 | Eval Round 3 | 评估删除/降级后的最终表现。 |

预期观察：

- 长期偏好与本次任务约束分层处理。
- “父亲不去 / 少步行不适用”不应永久污染后续家庭旅行。
- 删除“孩子喜欢动物”后，南京半日游不应继续强套动物园。

### 3. 考试复习计划

模块：学习偏好与冲刺模式模块

| 步骤 | 标签 | 输入 / 动作 |
| --- | --- | --- |
| 1 | Round 1 任务 | 帮我做一个 7 天英语复习计划。 |
| 2 | 形成记忆 | 以后学习计划请按 25 分钟番茄钟安排；我喜欢先看例题再讲知识点；每天最后要有 5 道自测题。 |
| 3 | 展示记忆 | 展示当前记忆。 |
| 4 | Eval Round 1 | 先评估当前 session，再进入下一轮。 |
| 5 | Round 2 应用 | 帮我做一个 5 天高数复习计划。 |
| 6 | 更新/条件化 | 考试只剩两天了，番茄钟不用了，改成按高频考点冲刺；但例题先行还保留。 |
| 7 | Eval Round 2 | 评估第二个 session 的费力度和记忆复用。 |
| 8 | Round 3 复用 | 帮我做 2 天线性代数冲刺。 |
| 9 | 删除后复测 | 删除每天 5 道自测题这条记忆。。然后：做一个物理复习计划。 |
| 10 | Eval Round 3 | 评估删除/降级后的最终表现。 |

预期观察：

- 能复用“例题先行”等学习偏好。
- 能识别“只剩两天”带来的冲刺模式变化。
- 删除自测题后，不再默认每天 5 道自测题。

### 4. 文献综述与研究设计

模块：研究方法模板与模式切换模块

| 步骤 | 标签 | 输入 / 动作 |
| --- | --- | --- |
| 1 | Round 1 任务 | 帮我整理一段关于多模态检索的文献综述。 |
| 2 | 形成记忆 | 以后做文献综述时，请按方法类别组织；每篇都标数据集、局限和可复现性；不要夸大结论。 |
| 3 | 展示记忆 | 展示当前记忆。 |
| 4 | Eval Round 1 | 先评估当前 session，再进入下一轮。 |
| 5 | Round 2 应用 | 帮我综述 RAG 评测方法。 |
| 6 | 更新/条件化 | 如果是头脑风暴研究问题，不要用文献综述模板；只保留谨慎表述。 |
| 7 | Eval Round 2 | 评估第二个 session 的费力度和记忆复用。 |
| 8 | Round 3 复用 | 帮我 brainstorm 3 个 RAG 研究问题。 |
| 9 | 删除后复测 | 删除每篇都标可复现性这条记忆。。然后：再做一段简短综述。 |
| 10 | Eval Round 3 | 评估删除/降级后的最终表现。 |

预期观察：

- 文献综述任务会复用方法类别、数据集、局限、谨慎表述。
- brainstorm 任务不会机械套用综述模板。
- 删除可复现性要求后，简短综述不再强制每篇标可复现性。

## 二、女朋友场景新增验证例子

这一组用于突出复杂送礼场景的深度能力，不替换 Workbench 前端预设场景。

### 验证例 2.1：三轮记忆管理

目标：一组三轮对话中覆盖记忆加入、更新、删除和展示。

#### Round 1：形成基础记忆

新开 Session，输入：

```text
帮我给女朋友选个生日礼物。
```

继续输入：

```text
预算1000元左右；她喜欢紫色；如果是首饰，她喜欢玫瑰金；以前送过玫瑰金项链，送过的不要再送。
```

继续输入：

```text
展示当前记忆。
```

预期：

- 记录预算：1000 元左右。
- 记录颜色偏好：喜欢紫色。
- 记录条件偏好：如果是首饰，喜欢玫瑰金。
- 记录历史礼物：以前送过玫瑰金项链，后续不要重复。

#### Round 2：跨 Session 复用并更新约束

新开 Session，输入：

```text
那再帮我给女朋友选生日礼物做一个推荐。
```

继续输入：

```text
不是，我想换个非首饰品类。
```

继续输入：

```text
展示当前记忆。
```

预期：

- 不要求用户重复预算、颜色、已送历史。
- 新增或更新约束：这次不要首饰，换非首饰品类。
- 后续推荐应避开项链和首饰类。

#### Round 3：删除偏好并确认具体礼物

新开 Session，输入：

```text
那再帮我给女朋友选生日礼物做一个新的推荐。
```

继续输入：

```text
删除 她喜欢紫色。然后：再给一个不重复的礼物方向。
```

当系统给出候选后，输入一个具体选择。已验证可用例：

```text
我选拍立得相机配相册
```

继续输入：

```text
展示当前记忆
```

预期：

- “喜欢紫色”进入 deleted，不再参与推荐。
- “拍立得相机配相册”以 `decision/selected` 形式保存。
- 不应保存成“这个”“第一款”“礼物方向”等无用泛称。

### 验证例 2.2：跨 Session 召回已选礼物

前置条件：完成验证例 2.1，且已保存具体礼物，例如“拍立得相机配相册”。

新开 Session，输入：

```text
我之前给她选过什么？
```

预期：

- 系统应直接回答之前选过的具体礼物，例如：

```text
你之前给女朋友选的是拍立得相机配相册。
```

也可以输入：

```text
最近我已经送过女朋友什么礼物？
```

预期：

- 系统应同时考虑历史礼物和已选礼物。
- 不应只回答“玫瑰金项链”，也不应漏掉最近选定的礼物。

## 三、Workflow 自学习能力验证

目标：验证用户在任务过程中教给系统的处理规则，会以 workflow 记忆沉淀，并在后续短 query 中生效。

### 验证例 3.1：选定礼物后直接问销售渠道

前置条件：已经在礼物场景中选定一个具体礼物，例如：

```text
我选拍立得相机配相册
```

继续输入教学反馈：

```text
以后我选定礼物后，如果我直接问销售渠道，你就按已选礼物直接给购买渠道、下单注意点和包装建议，不要重新推荐礼物。
```

继续输入：

```text
展示当前记忆
```

预期：

- 当前记忆中应包含一条 `workflow/gift_planning` 记忆。
- 内容应表达：选定礼物后，如果用户直接问销售渠道，就按已选礼物给渠道、下单注意点和包装建议，不要重新推荐礼物。
- 已选礼物本身仍保持 active，不应被 workflow 教学误降级或替换。

新开 Session，输入：

```text
销售渠道
```

预期：

- 系统应识别这是围绕上次已选礼物的短续问。
- 系统应给出对应已选礼物的购买渠道、下单注意点和包装建议。
- 系统不应重新发散推荐其他礼物方向。

已验证输出形态示例：

```text
线上优先看京东自营和富士官方旗舰店，自营物流快、售后方便。
富士天猫旗舰店偶尔有礼盒装，包装比较体面，送人合适。
预算1000左右的话，富士 instax mini 系列入门款加上相册和几盒相纸基本够用。
```

## 四、验收记录

### Workbench 真实 LLM 验证

验证 provider：MiniMax

验证链路：

1. Reset Memory。
2. Reset Session。
3. 输入：`帮我给女朋友选生日礼物，预算1000元左右，不要首饰，给三个不重复方向。`
4. 输入：`我选拍立得相机配相册`
5. 输入：`以后我选定礼物后，如果我直接问销售渠道，你就按已选礼物直接给购买渠道、下单注意点和包装建议，不要重新推荐礼物。`
6. 输入：`展示当前记忆`
7. Reset Session。
8. 输入：`我之前给她选过什么？`
9. 输入：`销售渠道`

已确认：

- 记忆中 active 保存：
  - `给女朋友选礼物预算在 1000 元左右`
  - `给女朋友选礼物约束：不要首饰，换非首饰品类`
  - `本次给女朋友的礼物已选定为拍立得相机配相册`
  - `以后处理同类任务时，选定礼物后，如果我直接问销售渠道...不要重新推荐礼物`
- 新 Session 中，`我之前给她选过什么？` 能召回“拍立得相机配相册”。
- 新 Session 中，`销售渠道` 能围绕“拍立得相机配相册”回答购买渠道和下单建议。

### Agent Skill runtime 验证

验证后端：local JSON 临时目录

验证命令形态：

```bash
python3 -m assist_everything_betterandbetter_skill.cli memory-write \
  --allow-no-llm \
  --memory-dir "$TMP_DIR" \
  --memory-backend local \
  --context $'user: 帮我给女朋友选礼物。\nassistant: 推荐：拍立得相机配相册。' \
  '我选拍立得相机配相册'

python3 -m assist_everything_betterandbetter_skill.cli memory-write \
  --allow-no-llm \
  --memory-dir "$TMP_DIR" \
  --memory-backend local \
  --context $'user: 帮我给女朋友选礼物。\nassistant: 推荐：拍立得相机配相册。\nuser: 我选拍立得相机配相册' \
  '以后我选定礼物后，如果我直接问销售渠道，你就按已选礼物直接给购买渠道、下单注意点和包装建议，不要重新推荐礼物。'

python3 -m assist_everything_betterandbetter_skill.cli memory-pack \
  --allow-no-llm \
  --memory-dir "$TMP_DIR" \
  --memory-backend local \
  '销售渠道'
```

已确认：

- `memory-write` 能保存 `decision/selected`：`本次给女朋友的礼物已选定为拍立得相机配相册`。
- `memory-write` 能保存 `workflow`：选定礼物后，短问销售渠道时直接给渠道和下单建议，不再重新推荐。
- `memory-pack "销售渠道"` 能同时召回 selected gift 和 workflow 规则。
